1 Git and GitHub

In this section what is Git and GitHub will be explained.

Git is a free an open source distributed version control system. It was originally developed by Linus Tovvalds[1] to manage changes to the Linux kernel.
GitHub is an online service that uses Git to provide code repositories. GitHub has added functionality in comparison, and is one of the most popular code repository systems worldwide.
In other words Github is a platform that helps people solve different problems by building software together.

As it is seen in the picture below in any repository there is the main/master branch. This code could be anything from software in ATMs, to Game development, to farming.
In order to solve issues, bugs, or even upgrade features of software; changes in the main code need to happen. However, it would not be efficient if a team of developers
started updating and testing new features in the main code. This is be because the main code could activelly be connected to multiple platforms and a small change could
lead to a catastrophic failure of the whole system.

So the solution to that is version control!!
With this version control, branches are created from the main branch of the code. With this the users can now add new code, and test features without altering the main.
When changes are made on that branch the users can commit those changes to the branch/branches (multiple commits can happen).
Once the user/users feel that the changes are ready to be reviewed, then a pull request can be made. With this the team can now see the changes, review them and discuss, and ultimetly
merge the new code to the main. Useful link to understand what Git is can be found here.

Key termilogy in understanding Git and GitHub:

Repository
Is the container of all code
Branch
Is the separate working area for code under development
Clone
Is the connected copy of the repository locally
Fork
Is the copy of the repository locally
Commit
Is the addition of the code to the branch
Pull
Publish the committed changes to the repository to review and eventually merge
Merge
Is the merging from a branch into another

It is worth noting that you can work with repositories online, but also locally. The preferred way is to work on a repository locally and use the internet connection
for clone, fork, fetch, pull, and merge.

There are 4 main ways in creating and working with repositories

  1. Through the GitHub online interface
  2. Through terminals
  3. Through IDEs
  4. A combination of the above

2 Unix Shell

A Unix shell is a command-line interpreter or shell that provides a command line user interface for Unix-like operating systems.
The shell is both a Command Line Interface (CLI) and a scripting language, and is used by the operating system to control the execution of the system using shell scripts.[2]
In the shell it is less obvious how to perform tasks but allows powerful automation and is useful for any programmer in managing files and automating basic tasks.
The shell used below is GitBash. You can see from the picture below that the UI is a black box, where the user can input commands to control the operating system.

The syntax of the scripting language follows the following Command Flag/Option/Switch Argument.

One of the basic commands is the ls, which is used to list the files in the directory. Using the ls -a all files are listed, including the hidden items.
In order to show the directory the user is at the pwd command is used, whereas to change directory it is cd foldername


To create new directory is mkdir directoryname. To create new file is touch name.extension and to delete a file is rm name.extension

In order to understand directories better, the picture below depicts an example root directory structure. The shell starts at the root directory and when using the cd
command it goes one layer down. So is for example you are at the work folder and want to go to carol folder, then you can use cd ... The .. represents one folder above.
If you wanted to go two folders above then cd ../.. would be used. This is a fast way to work your way through different folders without having to type the whole
path all the time

Useful shell commands:

  • ls or ls -a , lists files in a directory
  • pwd print working directory
  • cd or cd .. , change directory
  • mkdir directoryname create a new directory
  • touch filename.extension create new file
  • rm filename.extension deletes file, and rm -r directoryname deletes a directory
  • Some useful flags --help , provides manual of command used -i , usually used with rm to be asked if you want to delete
